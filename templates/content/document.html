API Fundamentals Explained

What is an API?

Think of an API as a waiter in a restaurant:

  • You (Client) - The customer who wants food
  • API (Waiter) - Takes your order to the kitchen
  • Kitchen (Server) - Prepares the food
  • Food (Response) - Delivered back to you

You don't need to know how the kitchen works, just how to order through the waiter.

API Communication Flow

1
Your App Sends Request
"Get me weather for New York"
2
API Processes Request
Validates and routes to weather service
3
Returns Response
JSON data with temperature, conditions
4
Your App Uses Data
Displays weather to user

weather.py - Python Implementation 
import os
import requests
from typing import Dict, Optional

class WeatherAPI:
    def __init__(self):
        self.api_key = os.environ.get('WEATHER_API_KEY')
        self.base_url = "https://api.openweathermap.org/data/2.5"
    
    def get_weather(self, city: str) -> Optional[Dict]:
        """Fetch weather data for a city"""
        try:
            response = requests.get(
                f"{self.base_url}/weather",
                params={
                    "q": city,
                    "appid": self.api_key,
                    "units": "metric"
                },
                timeout=10  # 10 second timeout
            )
            response.raise_for_status()
            data = response.json()
            
            return {
                "city": data.get("name"),
                "temperature": data["main"]["temp"],
                "feels_like": data["main"]["feels_like"],
                "humidity": data["main"]["humidity"],
                "condition": data["weather"][0]["description"],
                "icon": data["weather"][0]["icon"]
            }
            
        except requests.exceptions.RequestException as e:
            print(f"Error fetching weather: {e}")
            return None

# Usage
if __name__ == "__main__":
    weather = WeatherAPI()
    result = weather.get_weather("London")
    if result:
        print(f"Weather in {result['city']}: {result['temperature']}¬∞C")

Step 5: Handle Errors & Edge Cases

Production-ready code must handle failures gracefully:

Error Handling in JavaScript 
async function getWeatherWithRetry(city, maxRetries = 3) {
    for (let attempt = 1; attempt <= maxRetries; attempt++) {
        try {
            const response = await fetch(
                `https://api.openweathermap.org/data/2.5/weather?q=${city}&appid=${API_KEY}`
            );
            
            // Handle different status codes
            if (response.status === 404) {
                throw new Error(`City "${city}" not found`);
            }
            
            if (response.status === 401) {
                throw new Error('Invalid API key');
            }
            
            if (response.status === 429) {
                // Rate limited - wait and retry
                const retryAfter = response.headers.get('Retry-After') || 60;
                await new Promise(resolve => 
                    setTimeout(resolve, retryAfter * 1000)
                );
                continue;
            }
            
            if (!response.ok) {
                throw new Error(`API error: ${response.status}`);
            }
            
            return await response.json();
            
        } catch (error) {
            console.error(`Attempt ${attempt} failed:`, error.message);
            
            if (attempt === maxRetries) {
                // Return cached data or default values
                return getCachedWeather(city) || {
                    city,
                    temperature: null,
                    error: 'Service unavailable'
                };
            }
            
            // Exponential backoff
            await new Promise(resolve => 
                setTimeout(resolve, 1000 * Math.pow(2, attempt))
            );
        }
    }
}

Common Errors to Handle:

  • 401 Unauthorized: Invalid/expired API key
  • 404 Not Found: Resource doesn't exist
  • 429 Too Many Requests: Rate limit exceeded
  • 500 Internal Error: API server issues
  • Network Errors: No internet connection

Solutions:

  • Implement retry logic with backoff
  • Cache responses for offline use
  • Use fallback APIs when possible
  • Show user-friendly error messages
  • Log errors for debugging

class WeatherDashboard {
    constructor(apiKey) {
        this.apiKey = apiKey;
        this.cache = new Map();
        this.cacheDuration = 10 * 60 * 1000; // 10 minutes
    }
    
    async getCurrentWeather(city) {
        // Check cache first
        const cacheKey = `current_${city}`;
        if (this.isCached(cacheKey)) {
            return this.getFromCache(cacheKey);
        }
        
        // API call
        const response = await fetch(
            `https://api.openweathermap.org/data/2.5/weather?q=${city}&appid=${this.apiKey}&units=metric`
        );
        
        const data = await response.json();
        
        // Transform data for frontend
        const weatherData = {
            city: data.name,
            temperature: Math.round(data.main.temp),
            feelsLike: Math.round(data.main.feels_like),
            humidity: data.main.humidity,
            windSpeed: data.wind.speed,
            condition: data.weather[0].main,
            description: data.weather[0].description,
            icon: `https://openweathermap.org/img/wn/${data.weather[0].icon}@2x.png`,
            sunrise: new Date(data.sys.sunrise * 1000).toLocaleTimeString(),
            sunset: new Date(data.sys.sunset * 1000).toLocaleTimeString()
        };
        
        // Cache the result
        this.cache.set(cacheKey, {
            data: weatherData,
            timestamp: Date.now()
        });
        
        return weatherData;
    }
    
    async getForecast(city) {
        // Similar implementation for 5-day forecast
    }
    
    isCached(key) {
        if (!this.cache.has(key)) return false;
        const cached = this.cache.get(key);
        return Date.now() - cached.timestamp < this.cacheDuration;
    }
    
    getFromCache(key) {
        return this.cache.get(key).data;
    }
}

Complete API Server - server.js 
const express = require('express');
const cors = require('cors');
const helmet = require('helmet');
const rateLimit = require('express-rate-limit');
const { v4: uuidv4 } = require('uuid');

const app = express();
const PORT = process.env.PORT || 3000;

// Middleware
app.use(helmet());
app.use(cors());
app.use(express.json());

// Rate limiting
const limiter = rateLimit({
    windowMs: 15 * 60 * 1000, // 15 minutes
    max: 100, // Limit each IP to 100 requests per windowMs
    message: 'Too many requests from this IP'
});
app.use('/api/', limiter);

// API Key validation middleware
const validateApiKey = (req, res, next) => {
    const apiKey = req.headers['x-api-key'] || req.query.api_key;
    
    if (!apiKey) {
        return res.status(401).json({
            error: 'API key required',
            code: 'MISSING_API_KEY'
        });
    }
    
    // In production, validate against database
    if (apiKey !== process.env.VALID_API_KEY) {
        return res.status(401).json({
            error: 'Invalid API key',
            code: 'INVALID_API_KEY'
        });
    }
    
    next();
};

// Apply API key validation to all routes
app.use('/api/v1/*', validateApiKey);

// Sample data store (use database in production)
const stocks = {
    'AAPL': { symbol: 'AAPL', price: 175.25, change: 1.25 },
    'GOOGL': { symbol: 'GOOGL', price: 135.80, change: -0.50 },
    'MSFT': { symbol: 'MSFT', price: 330.15, change: 2.75 }
};

// API Routes
app.get('/api/v1/stocks', (req, res) => {
    const { symbol } = req.query;
    
    if (symbol) {
        const stock = stocks[symbol.toUpperCase()];
        if (!stock) {
            return res.status(404).json({
                error: 'Stock not found',
                code: 'STOCK_NOT_FOUND'
            });
        }
        return res.json(stock);
    }
    
    // Return all stocks
    res.json(Object.values(stocks));
});

app.get('/api/v1/stocks/:symbol/history', (req, res) => {
    const { symbol } = req.params;
    const { days = 7 } = req.query;
    
    // Generate mock historical data
    const history = [];
    const basePrice = stocks[symbol]?.price || 100;
    
    for (let i = days; i >= 0; i--) {
        const date = new Date();
        date.setDate(date.getDate() - i);
        
        // Random price fluctuation
        const fluctuation = (Math.random() - 0.5) * 10;
        const price = basePrice + fluctuation;
        
        history.push({
            date: date.toISOString().split('T')[0],
            price: parseFloat(price.toFixed(2)),
            volume: Math.floor(Math.random() * 1000000) + 100000
        });
    }
    
    res.json({
        symbol: symbol.toUpperCase(),
        history
    });
});

// Generate API key endpoint (protected by admin key)
app.post('/api/admin/generate-key', (req, res) => {
    const adminKey = req.headers['x-admin-key'];
    
    if (adminKey !== process.env.ADMIN_KEY) {
        return res.status(403).json({ error: 'Forbidden' });
    }
    
    const newApiKey = `sk_live_${uuidv4().replace(/-/g, '')}`;
    
    // In production, save to database
    console.log('Generated new API key:', newApiKey);
    
    res.json({
        apiKey: newApiKey,
        message: 'Save this key securely',
        createdAt: new Date().toISOString()
    });
});

// Error handling middleware
app.use((err, req, res, next) => {
    console.error(err.stack);
    res.status(500).json({
        error: 'Internal server error',
        requestId: req.id,
        code: 'INTERNAL_ERROR'

});

// 404 handler
app.use('*', (req, res) => {

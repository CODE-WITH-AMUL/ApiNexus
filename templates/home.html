What is an API?

API stands for Application Programming Interface. It's a set of rules that allows different software applications to communicate with each other. APIs let you access data and functionality from other services without needing to know how they're implemented internally.

Authentication & API Keys

Most APIs require authentication to ensure only authorized users can access them. The most common method is using API keys.

What is an API Key?

An API key is a unique identifier that authenticates requests to an API. Think of it like a password that grants you access to the API's functionality. Each key is associated with your account and has specific permissions and rate limits.

Generating Your API Key

To generate an API key:

  1. Navigate to your API Explorer dashboard
  2. Go to "API Keys" in the sidebar
  3. Click "Create New API Key"
  4. Give your key a descriptive name (e.g., "Production Server" or "Development")
  5. Select the permissions your key needs
  6. Copy your key immediately - you won't be able to see it again!

Storing API Keys Securely

Important Security Warning

Never store API keys in client-side code, version control, or public repositories. Exposed API keys can be used by anyone and may result in unexpected charges or data breaches.

Best practices for storing API keys:

  • Use environment variables on your server
  • Store keys in secure secret management services
  • Use different keys for development and production
  • Rotate keys regularly (every 90 days recommended)
  • Set up IP restrictions when possible

Sending API Keys in Requests

API keys are typically sent in the request headers. Here's the standard format:

Authorization Header 
Authorization: Bearer YOUR_API_KEY_HERE

Some APIs may use different header names like X-API-Key or api_key. Always check the specific API's documentation for the correct format.

Making Requests

APIs typically use REST (Representational State Transfer) architecture. REST APIs use standard HTTP methods to perform operations on resources.

HTTP Method Purpose Example
GET Retrieve data Get user profile
POST Create new data Create new user
PUT Update existing data Update user details
DELETE Delete data Delete user account

Request Components

Headers

Metadata about the request including authentication, content type, and more.

Content-Type: application/json
Authorization: Bearer sk_123...

Query Parameters

Key-value pairs added to the URL for filtering, pagination, and sorting.

?limit=10&page=2&sort=date

Request Body

Data sent with POST, PUT, or PATCH requests, typically in JSON format.

{ "name": "John", "email": "john@example.com" }

Example Code

Here are practical examples of making API requests in different programming languages. These examples fetch user data from a sample API.

JavaScript (Fetch API)

JavaScript Example 
// Fetch users from API
async function fetchUsers() {
  const apiKey = process.env.API_KEY; // Store keys in environment variables
  
  try {
    const response = await fetch('https://api.example.com/v1/users', {
      method: 'GET',
      headers: {
        'Authorization': `Bearer ${apiKey}`,
        'Content-Type': 'application/json',
        'Accept': 'application/json'
      }
    });
    
    if (!response.ok) {
      throw new Error(`HTTP error! status: ${response.status}`);
    }
    
    const data = await response.json();
    console.log('Users:', data.users);
    return data;
  } catch (error) {
    console.error('Error fetching users:', error);
    throw error;
  }
}

// Create a new user
async function createUser(userData) {
  const response = await fetch('https://api.example.com/v1/users', {
    method: 'POST',
    headers: {
      'Authorization': `Bearer ${process.env.API_KEY}`,
      'Content-Type': 'application/json'
    },
    body: JSON.stringify(userData)
  });
  
  return await response.json();
}

Python (Requests Library)

Python Example 
import os
import requests
from typing import Dict, Any

API_KEY = os.environ.get('API_KEY')  # Store keys in environment variables
BASE_URL = 'https://api.example.com/v1'

def get_users(limit: int = 10, page: int = 1) -> Dict[str, Any]:
    """
    Fetch users from the API with pagination
    """
    headers = {
        'Authorization': f'Bearer {API_KEY}',
        'Content-Type': 'application/json',
        'Accept': 'application/json'
    }
    
    params = {
        'limit': limit,
        'page': page
    }
    
    try:
        response = requests.get(
            f'{BASE_URL}/users',
            headers=headers,
            params=params
        )
        response.raise_for_status()  # Raise exception for bad status codes
        return response.json()
    except requests.exceptions.RequestException as e:
        print(f'Error fetching users: {e}')
        raise

def create_user(user_data: Dict[str, Any]) -> Dict[str, Any]:
    """
    Create a new user via the API
    """
    headers = {
        'Authorization': f'Bearer {API_KEY}',
        'Content-Type': 'application/json'
    }
    
    response = requests.post(
        f'{BASE_URL}/users',
        headers=headers,
        json=user_data
    )
    response.raise_for_status()
    return response.json()

cURL Command Line

cURL Examples 
# Get users (replace YOUR_API_KEY with your actual key)
curl -X GET \
  'https://api.example.com/v1/users?limit=10&page=1' \
  -H 'Authorization: Bearer YOUR_API_KEY' \
  -H 'Content-Type: application/json' \
  -H 'Accept: application/json'

# Create a new user
curl -X POST \
  'https://api.example.com/v1/users' \
  -H 'Authorization: Bearer YOUR_API_KEY' \
  -H 'Content-Type: application/json' \
  -d '{
    "name": "John Doe",
    "email": "john@example.com",
    "role": "user"
  }'

# Update a user (replace :id with actual user ID)
curl -X PUT \
  'https://api.example.com/v1/users/:id' \
  -H 'Authorization: Bearer YOUR_API_KEY' \
  -H 'Content-Type: application/json' \
  -d '{
    "name": "John Smith",
    "email": "john.smith@example.com"
  }'

# Delete a user
curl -X DELETE \
  'https://api.example.com/v1/users/:id' \
  -H 'Authorization: Bearer YOUR_API_KEY'

Response Handling Tips

  • Always check the HTTP status code before processing response data
  • Implement proper error handling for different status codes
  • Log request IDs when provided for debugging
  • Check rate limit headers to avoid hitting limits
  • Handle network errors and timeouts gracefully

Best Practices

Rate Limiting

Most APIs implement rate limiting to prevent abuse and ensure fair usage. You'll typically encounter these limits:

Requests Per Minute

Limit on how many requests you can make in a 60-second window.

60-1000

Daily Limits

Total requests allowed per day, often higher for paid plans.

10,000+

Burst Limits

Short-term limits to prevent sudden spikes in traffic.

5-10/sec

Rate Limit Headers

Most APIs include rate limit information in response headers. Check for headers like:

X-RateLimit-Limit: 1000
X-RateLimit-Remaining: 850
X-RateLimit-Reset: 1625097600
Retry-After: 60

Security Best Practices

Do's

  • Store API keys in environment variables
  • Use different keys for different environments
  • Rotate keys regularly (every 90 days)
  • Implement request timeouts
  • Validate all API responses
  • Use HTTPS for all requests
  • Monitor your API usage

Don'ts

  • Never commit API keys to version control
  • Don't hardcode keys in client-side code
  • Avoid sharing keys across team members
  • Don't ignore rate limit headers
  • Avoid making synchronous API calls in UI threads
  • Don't assume APIs are always available
  • Avoid excessive retries without backoff

Implementation Tips

Implement Retry Logic

APIs can fail temporarily. Implement exponential backoff retry logic for transient failures (5xx errors, network issues).

Cache Responses

Cache API responses when appropriate to reduce load and improve performance. Respect cache-control headers.

Monitor and Log

Track API usage, error rates, and response times. Log request IDs for debugging production issues.

Frequently Asked Questions

1. What if I lose my API key?

You can regenerate a new API key from your dashboard. The old key will be immediately deactivated. Always update your applications with the new key before regenerating if possible.

2. How do I know if I'm approaching my rate limit?

Check the X-RateLimit-Remaining header in API responses. Most APIs also return a Retry-After header when you hit the limit.

3. Can I use APIs in my mobile app?

Yes, but you must never embed API keys directly in mobile apps. Instead, create a backend service that makes API calls on behalf of your app, or use mobile SDKs with proper authentication flows.

4. What's the difference between test and live API keys?

Test keys work with sandbox environments and don't affect real data or incur charges. Live keys interact with production systems. Always use test keys during development.

5. How do I handle API versioning?

Most APIs include versioning in the URL path (e.g., /v1/). When APIs update to new versions, you'll have time to migrate. Never assume APIs won't change.

6. What should I do if I get a 5xx error?

5xx errors indicate server-side issues. Implement retry logic with exponential backoff. If the issue persists, check the API status page or contact support with the request ID from the error response.
